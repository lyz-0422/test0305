Unit 04 Vue 3 Tutorials
VUE · 01Vue 3 核心概念
為什麼選 Vue 3？

Vue 3 相較 Vue 2 引入了 Composition API，讓邏輯可以按「功能」組織而非按「選項」分散，搭配 TypeScript 支援更完善，打包體積更小（Tree-shaking 友好）。

特性Options API（Vue 2 風格）Composition API（Vue 3 推薦）
邏輯組織分散在 data/methods/computed按功能集中在 setup()
TypeScript支援有限原生支援
邏輯複用Mixins（易衝突）Composables（清晰無衝突）
學習曲線較平緩稍陡但更靈活
專案建立 · Vite + Vue 3

使用 Vite 建立 Vue 3 專案是目前最推薦的方式，速度極快，開發體驗極佳。

# 建立專案 npm create vite@latest my-app -- --template vue cd my-app npm install npm run dev # 目錄結構 my-app/ ├── src/ │ ├── App.vue # 根元件 │ ├── main.js # 入口 │ ├── components/ # 元件 │ ├── composables/ # 可複用邏輯 │ ├── stores/ # Pinia │ └── router/ # Vue Router └── vite.config.js
SFC · Single File Component

Vue 的 .vue 檔案由三個區塊組成，清晰分離結構、邏輯、樣式。

<script setup> // Composition API — 最推薦寫法 import { ref, computed, onMounted } from 'vue' const count = ref(0) // 響應式基本型別 const doubled = computed(() => count.value * 2) // 計算屬性 onMounted(() => console.log('元件掛載完成')) function increment() { count.value++ } </script> <template> <div> <p>Count: {{ count }} · Doubled: {{ doubled }}</p> <button @click="increment">+1</button> </div> </template> <style scoped> /* scoped 樣式只影響此元件 */ button { padding: .5rem 1rem; } </style>
VUE · 02響應式系統 · ref / reactive / watch
ref vs reactive
import { ref, reactive, watch, watchEffect } from 'vue' // ref → 基本型別 or 物件（.value 存取） const name = ref('LYZ') name.value = 'New Name' // reactive → 物件（無需 .value） const state = reactive({ count: 0, loading: false }) state.count++ // watch → 監聽特定值的變化 watch(count, (newVal, oldVal) => { console.log(`\${oldVal} → \${newVal}`) }) // watchEffect → 自動追蹤依賴 watchEffect(() => console.log(count.value))

FB · 02Firestore CRUD 操作
新增 · 讀取 · 更新 · 刪除
import { db } from './firebase' import { collection, doc, addDoc, getDoc, getDocs, updateDoc, deleteDoc, onSnapshot, query, where, orderBy, serverTimestamp } from 'firebase/firestore' // ✅ 新增文件 const docRef = await addDoc(collection(db, 'todos'), { title: '學習 Vue 3', done: false, createdAt: serverTimestamp() }) // 📖 讀取單一文件 const snap = await getDoc(doc(db, 'todos', docRef.id)) if (snap.exists()) console.log(snap.data()) // 📖 讀取集合 + 查詢條件 const q = query( collection(db, 'todos'), where('done', '==', false), orderBy('createdAt', 'desc') ) const querySnap = await getDocs(q) querySnap.forEach(doc => console.log(doc.id, doc.data())) // ✏️ 更新 await updateDoc(doc(db, 'todos', docRef.id), { done: true }) // 🗑️ 刪除 await deleteDoc(doc(db, 'todos', docRef.id)) // 🔴 即時監聽（Real-time） const unsubscribe = onSnapshot(collection(db, 'todos'), snap => { todos.value = snap.docs.map(d => ({ id: d.id, ...d.data() })) }) onUnmounted(unsubscribe) // 元件卸載時取消監聽
💡 Firestore 資料結構：Collection（集合）→ Document（文件）→ Field（欄位）。每個文件是一個類 JSON 物件，文件 ID 由 Firebase 自動生成或自訂。

Unit 06 Composables · 可複用邏輯
VUE · 06Composables 設計模式
什麼是 Composable？

Composable 是一個以 use 為命名前綴的函式，封裝帶有狀態的可複用邏輯。它是 Vue 3 取代 Mixin 的最佳方式，清晰、無命名衝突、易於測試。

📁 約定放在 src/composables/ 目錄，命名為 useXxx.js
useTodos — Firebase Todo Composable
// src/composables/useTodos.js import { ref } from 'vue' import { db } from '../firebase' import { collection, addDoc, updateDoc, deleteDoc, doc, onSnapshot, orderBy, query, serverTimestamp } from 'firebase/firestore' export function useTodos() { const todos = ref([]) const loading = ref(false) // 即時監聽 const q = query(collection(db, 'todos'), orderBy('createdAt', 'desc')) const unsub = onSnapshot(q, snap => { todos.value = snap.docs.map(d => ({ id: d.id, ...d.data() })) }) async function addTodo(title) { await addDoc(collection(db, 'todos'), { title, done: false, createdAt: serverTimestamp() }) } async function toggleTodo(id, done) { await updateDoc(doc(db, 'todos', id), { done: !done }) } async function deleteTodo(id) { await deleteDoc(doc(db, 'todos', id)) } return { todos, loading, addTodo, toggleTodo, deleteTodo, unsub } }
useAuth — 認證狀態 Composable
// src/composables/useAuth.js import { ref, onMounted, onUnmounted } from 'vue' import { auth } from '../firebase' import { onAuthStateChanged, signOut } from 'firebase/auth' export function useAuth() { const user = ref(null) const loading = ref(true) let unsubscribe onMounted(() => { unsubscribe = onAuthStateChanged(auth, u => { user.value = u loading.value = false }) }) onUnmounted(() => unsubscribe?.()) const logout = () => signOut(auth) return { user, loading, logout } }
其他常用 Composables 模式
// useFetch — 通用 API 請求 export function useFetch(url) { const data = ref(null) const error = ref(null) const loading = ref(true) fetch(url).then(r => r.json()) .then(d => { data.value = d; loading.value = false }) .catch(e => { error.value = e; loading.value = false }) return { data, error, loading } } // useLocalStorage — 持久化狀態 export function useLocalStorage(key, defaultVal) { const val = ref(JSON.parse(localStorage.getItem(key) ?? 'null') ?? defaultVal) watch(val, v => localStorage.setItem(key, JSON.stringify(v)), { deep: true }) return val }
Unit 07 Todo Application 實戰
VUE · 07Vue 3 + Firestore Todo App 架構
完整 TodoApp.vue
<script setup> import { ref, computed } from 'vue' import { useTodos } from '../composables/useTodos' const { todos, addTodo, toggleTodo, deleteTodo } = useTodos() const input = ref('') const filter = ref('all') // 'all' | 'active' | 'done' const filtered = computed(() => filter.value === 'all' ? todos.value : filter.value === 'active' ? todos.value.filter(t => !t.done) : todos.value.filter(t => t.done) ) const doneCount = computed(() => todos.value.filter(t => t.done).length) const activeCount = computed(() => todos.value.filter(t => !t.done).length) function handleAdd() { if (!input.value.trim()) return addTodo(input.value.trim()) input.value = '' } </script> <template> <div class="todo-app"> <h1>待辦清單 ({{ activeCount }} 項待完成)</h1> <!-- 新增 --> <form @submit.prevent="handleAdd"> <input v-model="input" placeholder="新增待辦..."/> <button type="submit">新增</button> </form> <!-- 篩選 --> <div class="filters"> <button v-for="f in ['all','active','done']" :key="f" :class="{ active: filter===f }" @click="filter=f"> {{ f }}</button> </div> <!-- 清單 --> <TransitionGroup name="list" tag="ul"> <li v-for="todo in filtered" :key="todo.id"> <input type="checkbox" :checked="todo.done" @change="toggleTodo(todo.id, todo.done)"/> <span :class="{ done: todo.done }">{{ todo.title }}</span> <button @click="deleteTodo(todo.id)">刪除</button> </li> </TransitionGroup> </div> </template>
✦ 重點：TransitionGroup 為清單項目加入進出動畫；computed 實現不同篩選條件；Firestore onSnapshot 實現多裝置即時同步。

Unit 13 Pinia · 狀態管理
PN · 01Pinia 設計與使用
什麼是 Pinia？為什麼用它？

Pinia 是 Vue 3 官方推薦的狀態管理庫（Vuex 的繼任者）。它更輕量、API 更簡潔、完整 TypeScript 支援、沒有 mutation 的繁瑣限制。

📊 使用時機：多個元件需要共用同一份狀態；跨路由頁面傳遞資料；需要持久化的全域狀態（登入使用者、購物車、主題設定）。
安裝與建立 Store
npm install pinia // main.js import { createPinia } from 'pinia' app.use(createPinia()) // src/stores/auth.js — Setup Store 寫法（推薦） import { ref, computed } from 'vue' import { defineStore } from 'pinia' import { auth } from '../firebase' import { onAuthStateChanged, signOut } from 'firebase/auth' export const useAuthStore = defineStore('auth', () => { // State const user = ref(null) const loading = ref(true) // Getters const isLoggedIn = computed(() => !!user.value) const displayName = computed(() => user.value?.displayName ?? '訪客') // Actions onAuthStateChanged(auth, u => { user.value = u; loading.value = false }) const logout = () => signOut(auth) return { user, loading, isLoggedIn, displayName, logout } })
在元件中使用 Store
<script setup> import { storeToRefs } from 'pinia' import { useAuthStore } from '../stores/auth' const authStore = useAuthStore() // storeToRefs 保持響應性（直接解構會失去） const { user, isLoggedIn, displayName } = storeToRefs(authStore) // Actions 可直接解構（函式不需要 storeToRefs） const { logout } = authStore </script> <template> <div v-if="isLoggedIn"> 歡迎，{{ displayName }} <button @click="logout">登出</button> </div> </template>
持久化 Store — pinia-plugin-persistedstate
npm install pinia-plugin-persistedstate // main.js import piniaPluginPersistedstate from 'pinia-plugin-persistedstate' const pinia = createPinia() pinia.use(piniaPluginPersistedstate) // store — 加上 persist: true 即自動存 localStorage export const useSettingsStore = defineStore('settings', () => { const theme = ref('light') const lang = ref('zh') return { theme, lang } }, { persist: true })
Unit 14 Vercel · 快速部署
VRC · 01Vercel 部署流程
為什麼選 Vercel？

Vercel 是目前最流行的靜態網站與前端框架部署平台，免費方案即可使用，支援 GitHub 自動部署（每次 push 自動更新）、自訂域名、Preview Deployments、環境變數管理。

特性說明
自動部署連結 GitHub repo，每次 push 自動觸發部署
Preview URL每個 PR 都有獨立預覽連結
環境變數Firebase 金鑰等敏感資訊安全存放
Global CDN全球加速，台灣訪問也很快
免費額度個人專案完全夠用
方法一：GitHub 連動（推薦）
1. 前往 vercel.com → 登入（用 GitHub） 2. "New Project" → 選擇你的 GitHub repo 3. Vercel 自動偵測 Vite → Build Command: npm run build 4. Output Directory: dist（Vite 預設） 5. 點擊 Deploy → 幾十秒內完成部署 6. 取得 *.vercel.app 網址，可立即分享
方法二：CLI 部署
# 安裝 Vercel CLI npm install -g vercel # 登入 vercel login # 在專案目錄執行（第一次會問設定） vercel # 部署到 production vercel --prod
環境變數設定（Firebase 金鑰安全存放）
# .env.local（本地用，加入 .gitignore！） VITE_FIREBASE_API_KEY=your-api-key VITE_FIREBASE_PROJECT_ID=your-project-id // src/firebase.js — 讀取環境變數 const firebaseConfig = { apiKey: import.meta.env.VITE_FIREBASE_API_KEY, projectId: import.meta.env.VITE_FIREBASE_PROJECT_ID, } # Vercel Dashboard → Settings → Environment Variables # 將 .env.local 的內容複製進去
