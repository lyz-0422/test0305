VUE 3 TUTORIALS
Options vs Composition API
›

Vue 3 提供兩種寫法。Options API 以物件結構組織，適合初學；Composition API 用 setup() 集中邏輯，適合大型專案複用。

setup()ref()reactive()computed()
// Options API export default { data() { return { count: 0 } }, methods: { inc() { this.count++ } } } // Composition API（推薦） import { ref, computed } from 'vue' const count = ref(0) const double = computed(() => count.value * 2) const inc = () => count.value++

COMPOSABLES
可複用的邏輯組合函式
›

Composables 是以 use 開頭的函式，將響應式邏輯封裝成可複用單元。等同於 React Hooks，是 Composition API 的精髓所在。

// useCounter.js — 最簡單的 composable import { ref } from 'vue' export function useCounter(init = 0) { const count = ref(init) const inc = () => count.value++ const dec = () => count.value-- const reset = () => count.value = init return { count, inc, dec, reset } } // 在任何元件中使用 const { count, inc } = useCounter(10)

    FIREBASE
    後端即服務 · BaaS
    ›

    Firebase 是 Google 提供的後端服務平台，提供資料庫、身份驗證、雲端儲存等功能，讓前端開發者不需寫後端即可建立完整應用。

    AuthenticationFirestoreStorageHosting
    // firebase.js — 初始化 import { initializeApp } from 'firebase/app' import { getAuth } from 'firebase/auth' import { getFirestore } from 'firebase/firestore' const app = initializeApp({ apiKey: "AIza...", authDomain: "project.firebaseapp.com", projectId: "my-project", }) export const auth = getAuth(app) export const db = getFirestore(app) // Google 登入 import { GoogleAuthProvider, signInWithPopup } from 'firebase/auth' const login = () => signInWithPopup(auth, new GoogleAuthProvider())

    05 · 02
    FIRESTORE
    NoSQL 即時資料庫 CRUD
    ›

    Firestore 以 Collection → Document → Field 層級組織資料。支援即時監聽（onSnapshot），資料變動時自動更新 UI，無需輪詢。

    collectionaddDocupdateDocdeleteDoconSnapshot
    import { collection, addDoc, getDocs, updateDoc, deleteDoc, doc, onSnapshot, query, orderBy } from 'firebase/firestore' // 新增文件 await addDoc(collection(db, 'todos'), { text: '學 Firestore', done: false, createdAt: serverTimestamp() }) // 即時監聽（onSnapshot） const q = query(collection(db, 'todos'), orderBy('createdAt')) onSnapshot(q, snapshot => { todos.value = snapshot.docs.map(d => ({ id: d.id, ...d.data() })) })

    TAILWIND CSS
    Utility-First CSS 框架
    ›

    Tailwind 以原子化 class 取代手寫 CSS，直接在 HTML 上組合樣式。不需命名 class，不會有樣式衝突，build 時自動移除未用 class。

    PINIA
    Vue 3 官方狀態管理
    ›

    Pinia 是 Vuex 的繼任者，Vue 3 官方推薦的全域狀態管理庫。以 defineStore 定義 Store，比 Vuex 更簡潔，完整支援 TypeScript 與 DevTools。

    defineStorestategettersactionsstoreToRefs
    // stores/useCounterStore.js import { defineStore } from 'pinia' import { ref, computed } from 'vue' export const useCounterStore = defineStore('counter', () => { // state const count = ref(0) const name = ref('LYZ') // getters const double = computed(() => count.value * 2) // actions const increment = () => count.value++ const reset = () => count.value = 0 return { count, name, double, increment, reset } }) // 在元件中使用 const store = useCounterStore() const { count, double } = storeToRefs(store) // 保持響應性

    07 · 02
    VERCEL
    快速部署 · 零設定上線
    ›

    Vercel 是最流行的前端部署平台，連結 GitHub 後每次 push 自動部署，自動 HTTPS、CDN 加速、Preview 預覽網址，完全免費開始使用。

    GitHub 整合自動 CI/CDPreview URLEdge Network
    # 1. 安裝 Vercel CLI npm install -g vercel # 2. 在專案目錄登入並部署 vercel # 首次部署（互動式設定） vercel --prod # 部署到正式環境 # 或直接從 GitHub 一鍵部署 # vercel.com → New Project → Import Git Repository # vite.config.js — Vite 專案不需額外設定 # vercel 自動偵測 npm run build → dist/
    Vercel 部署流程
    01
    npm create vue@latest my-app
    用 Vite 建立 Vue 3 專案，選擇 Router、Pinia
    02
    git init → git push → GitHub
    將專案推送到 GitHub repository
    03
    vercel.com → Import Project
    連結 GitHub Repo，Vercel 自動偵測框架設定
    04
    Push to main → 自動部署
    之後每次 push 自動觸發 build，Preview URL 即時生成
    05
    https://my-app.vercel.app ✓
    自動 HTTPS、全球 CDN、免費 custom domain 設定
